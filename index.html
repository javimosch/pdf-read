pdf-read

The agent-first CLI for blazing fast PDF text extraction with bulletproof OCR fallback.

Get Started

Why pdf-read?

Agent-Friendly

Designed for AI agents. JSON-first, stable schemas, and semantic exit codes.

Fast Native Engine

Uses pdftotext for near-instant extraction from text-based PDFs.

OCR Fallback

Scanned doc? No problem. Automatically falls back to Tesseract OCR when text is sparse.

Installation

npm install -g pdf-read

Ensure you have poppler-utils installed on your system:

# Ubuntu/Debian
sudo apt-get install poppler-utils tesseract-ocr

CLI Reference

Usage: pdf-read read [options] [file]

Option Description
-t, --text Output as plain text instead of JSON
--deep Force OCR extraction (slower but 100% text recovery)
-p, --pages <range> Page range (e.g., 1-5 or 1,3,5)
-v, --verbose Include rich metadata in output
--help-json Output help as machine-readable JSON

Semantic Exit Codes

Code Category Meaning
0 Success Operation completed successfully
80 User Error Invalid argument or syntax
91 Resource Error File not found
92 Resource Error Not a PDF file
93 Extraction Error No text could be extracted (even with OCR)
106 System Error OCR engine failed (Tesseract)
110 Software Error Internal software error / bug
